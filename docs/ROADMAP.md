# Development Philosophy

PathTree is developed incrementally.

Every milestone must result in a usable improvement.

Large features are divided into small pull requests.

The project follows one simple rule:

> Build a solid foundation first. Expand later.

---

# Version 0.0.1 — Foundation

Goal:

Create the technical foundation.

Features:

- Python project setup
- SQLite database
- Node model
- Repository layer
- Basic Textual application
- Tree widget
- Basic keyboard navigation
- Shell adapter (Bash/Zsh)

Release goal:

Navigate a tree and change directories.

---

# Version 0.0.2 — Daily Navigation

Goal:

Make PathTree useful for everyday navigation.

Features:

- Search
- Favorites
- Temporary entries
- Permanent bookmarks
- Category management
- Nested folders
- Keyboard improvements

Release goal:

Replace manual directory navigation during daily work.

---

# Version 0.0.3 — Actions

Goal:

Allow nodes to perform actions.

Features:

- Action model
- Launch commands
- Working directory
- Command arguments
- Multiple actions per node

Release goal:

A node can store and execute useful actions.

---

# Version 0.1.0 — First Stable Release

Goal:

A reliable application suitable for daily use.

Features:

- Stable tree navigation
- Search
- Favorites
- Temporary bookmarks
- Permanent bookmarks
- Actions
- Import / Export
- Backup
- Stable shell integration

Release goal:

A complete and dependable terminal workspace manager.

---

# After Version 0.1

The following features will be evaluated based on practical experience:

- Plugin system
- Git integration
- Docker integration
- SSH integration
- Variables
- Resource management
- Themes

These features are intentionally postponed until the core application has proven itself in daily use.
